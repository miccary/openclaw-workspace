# 跨境电商 AI 数据平台蓝图（个人/小团队版）

## 1) 目标与原则
- 目标：用低成本搭建“数据采集 → 数据仓库 → BI → AI分析 → 自动报告”闭环。
- 原则：
  - 先可用，再完善（MVP优先）
  - 低耦合模块化（可替换）
  - 自动化优先（定时 + 审计）
  - 成本可控（先CPU可跑）

---

## 2) 总体架构
数据源 → 采集层 → 存储层 → 处理层 → BI层 → AI Agent层 → 应用层

- 数据源：平台API、爬虫、广告后台、竞品页面
- 采集层：Python/Scrapy
- 存储层：PostgreSQL
- 处理层：Pandas ETL + SQL视图
- BI层：Power BI / Metabase
- AI层：OpenClaw（日报、预警、策略建议）
- 应用层：Telegram/钉钉推送、Notion归档

---

## 3) 里程碑（四阶段）

### Phase 1（第1-2周）MVP打底
- PostgreSQL建库建表
- 每日采集（销量/广告/库存）
- 首版ETL（去重、字段统一、异常值处理）
- 初版仪表板（店铺健康度）

### Phase 2（第3-4周）BI与稳定性
- 5个核心看板上线
- 定时任务稳定（重试/告警）
- 数据质量监控（空值率、延迟、重复率）

### Phase 3（第5-6周）AI分析闭环
- OpenClaw每日自动运营报告
- 风险预警（库存<15天、ACOS异常）
- 竞品变化摘要

### Phase 4（第7-8周）扩展与优化
- Listing优化建议
- 周报/月报自动化
- 模型路由与成本控制策略

---

## 4) 数据模型（6张核心表）

1. `products`（产品主数据）
2. `asins`（ASIN映射）
3. `sales_daily`（每日销售）
4. `ads_daily`（每日广告）
5. `competitor_data`（竞品快照）
6. `keyword_rank`（关键词排名）

> 建议补充：`etl_runs`（任务执行日志）和 `alerts`（预警事件），用于审计与复盘。

---

## 5) 每日数据流（标准作业）
1. 00:30 采集（API/爬虫）
2. 01:00 清洗与入库（ETL）
3. 01:30 刷新BI数据集
4. 08:30 AI生成晨报
5. 14:00 AI生成进展提醒
6. 21:30 AI生成复盘

---

## 6) BI看板（5个）
1. 店铺健康度（销售额、订单、TACOS、库存周转）
2. SKU盈利能力（销售、广告花费、毛利率、库存）
3. 广告效率（ACOS、CTR、CVR、广告订单占比）
4. 竞品监控（价格、排名、评论增速）
5. 风险预警（库存、评分、投放异常）

---

## 7) AI Agent最小闭环（先做这3个）
1. 自动日报：销售/广告/库存总结 + 风险 + 机会
2. 竞品分析：价格与评论变化、排名波动解释
3. Listing优化：评论高频痛点 + 关键词建议

输出渠道：Telegram/钉钉 + Notion归档。

---

## 8) 技术建议（当前版本）
- 采集与ETL：Python + Pandas
- 数据库：PostgreSQL
- 调度：先Cron，后续再上Airflow
- API层：FastAPI（可选）
- BI：Power BI（你熟）
- AI：OpenClaw（多模型路由，主Codex，轻任务DeepSeek）

---

## 9) 安全与治理（必须）
- API Key最小权限与定期轮换
- 采集与AI执行日志全量保留
- 临时提权 + 自动回收（TTL）
- 高风险动作二次确认
- 每周跑一次安全审计
